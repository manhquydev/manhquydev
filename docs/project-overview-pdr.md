# Project Overview & Product Development Requirements (PDR)

## Project Purpose
Transform the existing GitHub profile into a high-impact, bilingual technical portfolio that demonstrates expertise in Full-Stack development and EdTech innovation.

## Target Audience
- Technical Recruiters and Hiring Managers
- Potential Collaborators
- Students and Educators in the EdTech space

## Functional Requirements
- **Bilingual Support**: Full professional content in English with collapsible Vietnamese sections.
- **Impact Metrics**: Data-driven descriptions of projects (e.g., student counts, performance improvements).
- **Featured Projects**: Detailed breakdown of primary work including problem/solution/impact statements.
- **Dynamic Stats**: Real-time visualization of coding activity and repository statistics.
- **Call to Action (CTA)**: Clear paths for hiring, viewing resume, or contacting.

## Non-Functional Requirements
- **Performance**: High readability with optimized Markdown structure.
- **SEO**: Use of relevant keywords for EdTech and Full-Stack development.
- **Accessibility**: Clear visual hierarchy and semantic structure.
- **Maintainability**: Automated updates for blog posts and coding stats via GitHub Actions.

## Acceptance Criteria
- README.md must be bilingual (EN primary).
- All external links (Website, Resume, LinkedIn) must be functional.
- Impact metrics must be clearly visible above the fold.
- Project details must include both high-level summaries and tech stack specifics.

## Phase 1 Completion Status
- **Status**: 100% Complete
- **Deliverables**: Rewrite of README.md with bilingual content, featured projects section, and recruiter-optimized layout.
